# Governance Model

The Impact Collaboration Platform (ICP) is an open-source project dedicated to facilitating social impact connections. This document describes how the project is governed.

## Roles and Responsibilities

### Maintainers
Maintainers are responsible for the strategic direction of the project, including:
- Approving pull requests.
- Deciding on architectural changes.
- Managing releases.
- Enforcing the Code of Conduct.

### Contributors
Contributors are community members who:
- Submit pull requests.
- Report issues.
- Discuss ideas in issues.

Anyone can become a contributor by following the [Contributing Guidelines](./CONTRIBUTING.md).

## Decision Making Process

Decisions are generally made by consensus among maintainers.
- **Minor Changes**: Can be approved by a single maintainer.
- **Major Changes**: Require discussion (e.g., via an Issue or RFC) and consensus from the majority of maintainers.

## Project Roadmap
The project roadmap is managed via [GitHub Issues/Projects]. We prioritize features that align with UN Sustainable Development Goals (SDGs), particularly SDG 17 (Partnerships).
